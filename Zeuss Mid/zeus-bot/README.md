# Zaetheron Industry — бот + мини-апп + сервер ключей

## Структура
```
zeus-bot/
├── key_server.py     # FastAPI, отдаёт мини-апп и проверяет ключи (Postgres на Railway)
├── bot.py             # Telegram-бот (long polling)
├── webapp/index.html  # Mini App: тарифы / ключ / профиль
└── requirements.txt
```

## 1. Деплой сервера ключей (`key_server.py`) на Railway

Скорее всего у вас это уже развёрнуто — просто замените `key_server.py` на обновлённую версию
(добавлен `/check` и раздача `webapp/index.html` по адресу `/app`) и загрузите папку `webapp/`
рядом с ним в тот же сервис.

Переменные окружения этого сервиса:
- `DATABASE_URL` — уже стоит автоматически, если Postgres подключён к проекту
- `ZEUS_ADMIN_TOKEN` — токен для ручного `/deactivate`, замените на свой
- `CLIENT_DOWNLOAD_URL` — прямая ссылка на файл клиента (например, с вашего же хостинга или облака).
  Если не задать — кнопка «Скачать» просто не появится, даже при валидном ключе.

Start command сервиса: `uvicorn key_server:app --host 0.0.0.0 --port $PORT`

После деплоя мини-апп будет доступен по адресу вида:
`https://<ваш-сервис>.up.railway.app/app`
Он должен открываться в браузере уже сейчас — проверьте перед следующим шагом.

## 2. Бот (`bot.py`) — второй сервис в том же проекте Railway

В Railway: New Service → тот же репозиторий → Start Command:
```
python bot.py
```

Переменные окружения этого сервиса:
- `BOT_TOKEN` — получить у [@BotFather](https://t.me/BotFather) командой `/newbot`
- `ADMIN_CHAT_ID` — ваш личный Telegram ID числом. Узнать: напишите [@userinfobot](https://t.me/userinfobot)
- `WEBAPP_URL` — ссылка на мини-апп из шага 1, `https://<ваш-сервис>.up.railway.app/app`
- `DATABASE_URL` — та же Postgres, что у сервиса `key_server.py` (в Railway: тот же Postgres-плагин,
  подключите его и к сервису бота тоже — тогда переменная подставится автоматически)

### Добавление ключей прямо из бота

Пишете боту (в личку, от аккаунта с `ADMIN_CHAT_ID`):
```
/addkey ZEUS-ABCD-1234 30 заказ от @username
```
- `ZEUS-ABCD-1234` — придуманный вами текст ключа (любой, главное уникальный)
- `30` — срок действия в днях (0 = бессрочно)
- `заказ от @username` — необязательная заметка

Бот сразу добавит ключ в ту же базу, из которой мини-апп и клиент проверяют ключи —
никакого отдельного шага не нужно, ключ «регистрируется как есть».

Отозвать ключ: `/delkey ZEUS-ABCD-1234`

## 3. Настройка Mini App в @BotFather

1. `/mybots` → выберите бота → **Bot Settings → Menu Button**
2. Укажите тот же `WEBAPP_URL` — тогда кнопка мини-аппа появится прямо в поле ввода чата,
   не только по команде `/start`
3. Опционально: `/setdescription`, `/setuserpic` — для оформления карточки бота

## 4. Как работает продажа

Покупатель жмёт «Купить» на тарифе → мини-апп шлёт данные боту через `Telegram.WebApp.sendData` →
бот пересылает вам личное сообщение с тарифом, ценой и ссылкой на чат с покупателем.
Дальше вы вручную договариваетесь об оплате и выдаёте ключ через `manage_keys.py`:

```
python manage_keys.py add --days 30 --note "заказ от @username"
```

Ключ отправляете покупателю лично в чате — автоматической выдачи после оплаты в этой версии нет
(вы сами говорили, что хотите вручную вести цену/оплату).

## 5. Профиль в мини-аппе

Вкладка «Профиль» показывает имя/аватар из Telegram (`initDataUnsafe.user`) и статус ключа,
который пользователь один раз ввёл на вкладке «Ключ» (хранится у него в браузере, в `localStorage`
мини-аппа — сервер не хранит связку telegram_id → key, только сам ключ).
Там же — самостоятельный сброс привязки к устройству (использует `resets_left` из вашей таблицы `keys`).

## Дальше можно улучшить
- Автоматическая выдача ключа сразу после ручного подтверждения оплаты (кнопка в чате с админом)
- Привязка `telegram_id` к ключу в БД — тогда профиль будет подтягивать ключ сам, без ручного ввода
- Приём оплаты прямо в Telegram (Telegram Payments / ЮKassa-провайдер), без ручного «жду перевод»
